# CMMC L2 Gap Workshop Tool Roadmap after v76

## v76 — Responsibility Review UX — completed

- Added one-practice-at-a-time responsibility review for live advisory sessions.
- Combined practice requirement, current Workshop owners, service/module context, provider duties, client duties, responsibility dimensions, validation questions, actions, and advisor decision in one surface.
- Added previous/next, queue navigation, completion indicator, Alt+Left/Right navigation, and direct jump to related Practice Review.
- Moved the full responsibility matrix under Advanced / Bulk Review.
- Preserved the v75 separate-overlay model and all stable package contracts.

## v77 — Evidence Ownership and Provider Follow-up — committed next release

### Primary objective

Connect reviewed responsibility context to evidence production, retention, access, submission, and follow-up ownership without converting responsibility guidance into evidence sufficiency.

### Planned work

- Derive explicit working evidence-production, evidence-retention, evidence-access, and evidence-submission candidates from accepted responsibility records.
- Distinguish provider-produced platform evidence, client configuration evidence, client operational evidence, and shared/combined evidence.
- Show contract/SOW/RACI validation needs and actual evidence-access limitations.
- Generate deduplicated client and provider evidence requests from explicit advisor selections.
- Connect requests to the persistent Action & Blocker Register and existing Evidence Follow-up workflow.
- Preserve all evidence sufficiency, Met/Not Met, readiness, and assessment conclusions as human-controlled.

## v78 — Reporting and SSP/Workbook Alignment — committed

- Add reviewed responsibility and evidence-ownership summaries to client and advisor reports.
- Carry accepted context to SSP and workbook helper surfaces where contract-safe.
- Preserve official workbook results and SSP authoring status as reviewer-controlled overlays.

## v79 — Full McFirecoal Toolchain Regression — committed

- Run DocConverter, Scoper, Workshop, Builder/Merger, SSP, and Control Center through the complete McFirecoal package.
- Validate package routing, version drift, source lineage, actions, Workbook return, SSP round trip, responsibility, and evidence ownership.
- Publish a named validated suite snapshot.

## v80 — Stabilization and Contract Freeze — committed

- Complete full 110-practice and 320-objective regression.
- Validate accessibility, mobile, print, dark/light, performance, recovery, and backward compatibility.
- Freeze the first production responsibility-reconciliation contract when supported by regression evidence.

## v81 — Contract and RACI Mapping — proposed

- Add structured import/review for responsibility matrices, contracts, SOWs, SLAs, and provider control matrices.
- Map clauses and rows to services, responsibility dimensions, practices, questions, and provenance.

## v82 — Responsibility Evidence Pack Generator — proposed

- Generate provider-specific evidence request packs from accepted responsibility and evidence-ownership context.
- Separate platform attestations, customer configuration evidence, operational evidence, and shared evidence.

## v83 — Reusable Service Profile Library — proposed

- Support locally stored, engagement-independent service templates with version/date governance.
- Require engagement-specific confirmation before use.

## v84 — Portable Suite Readiness — proposed

- Optimize Workshop startup, state size, export size, and module packaging for the portable L2G suite.
- Validate relative paths, Control Center launch, manifests, hashes, update/rollback, and no-install operation.

## v85 — Production Suite Snapshot and Long-Term Support — proposed

- Publish a validated suite snapshot across all modules.
- Define supported contract/version ranges, migration policy, regression cadence, and long-term support rules.

## v86 — Responsibility Pack Maintenance Governance — proposed

- Add scheduled source-review records, pack retirement/replacement rules, and reference-age thresholds.
- Keep provider guidance current without converting official-reference material into final responsibility determinations.
