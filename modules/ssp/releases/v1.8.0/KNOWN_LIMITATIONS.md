# Known Limitations — v1.8.0

- Child modules are metadata shells; only the anchor has authoritative SSP content.
- Module requirement applicability and responsibility are not implemented.
- Inheritance, supplements, overrides, and parent-change impact are not implemented.
- Conflict detection and stale-parent workflows are not implemented.
- Consolidated/module CRM and module Word Review are not implemented.
- Approvals, immutable baselines, version comparison, and restoration are not implemented.
- Configurable customer validation rules are documented but not active.
- No multi-user backend, synchronization, cloud discovery, or direct external integration is included.

These limitations are deliberate release boundaries, not hidden acceptance claims.
