# Validation Rule Catalog — v1.8.0 Foundation

Active foundation rules: schema/version support, stable-ID format, one root anchor, unique module IDs, parent existence, required parent for children, circular-reference prevention, maximum 50 modules, required names/short names, allowed module types/statuses, and import text/list bounds.

Planned configurable rules for v1.8.7 include component-count reconciliation, authoritative identity-source conflicts, external-system boundary treatment, missing diagrams/data flows, undecided tooling, non-employee identity gaps, missing responsibility/evidence owners, inheritance without source, shared responsibility without residual duties, inconsistent cross-module status, and unreviewed parent changes.

Planned rules must report findings; they must not automatically reconcile contradictions or make compliance determinations.
