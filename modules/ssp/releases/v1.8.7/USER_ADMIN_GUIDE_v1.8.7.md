# User and Administrator Guide — v1.8.7

1. Continue using Single-System mode unless a portfolio is required.
2. In portfolio mode, complete formal review and approval for the desired portfolio/module scope.
3. Resolve open blocking conflicts or document an active authorized exception naming each conflict.
4. In **Immutable named baselines**, enter a unique name, version, description, and release notes, then create the baseline.
5. Select any two baselines to compare and export the full JSON difference report when needed.
6. Export individual baseline packages for controlled offline retention.
7. To restore, act as an assigned Administrator, select the baseline, enter a rationale, download a current backup, and restore. The tool creates a new working version and requires a new review/approval cycle.
8. Do not treat local role names, fingerprints, approvals, or baselines as digital signatures, assessment results, or certification decisions.
