# Security and Privacy Design — v1.8.7

All baseline work remains local and offline. Creation requires fresh approval, role scope, and conflict gating. Baseline records have no edit/delete UI. Restoration requires Administrator scope, rationale, and a backup prompt. Deterministic FNV-1a fingerprints detect local content differences but are explicitly not represented as cryptographic signatures. No telemetry, remote code, or external storage is introduced.
