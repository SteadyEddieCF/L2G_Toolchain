# Release Notes — CMMC L2 SSP v1.8.1

## Added

- Exactly 110 stable module-requirement records per module.
- Six applicability values, including Pending decision, shared responsibility, locally implemented, inherited declarations, supplements, and Not applicable.
- Responsibility model, CSP/ESP/organization/module duties, owners, evidence references, scope, remediation, POA&M, review, and approval-tracking fields.
- Individual decision editor, family/search/applicability filters, unresolved queue, and bounded bulk actions.
- Required rationale for Not applicable and required source metadata for inherited declarations.
- Automatic v1.8.0 portfolio upgrade without inferred decisions.
- Module exports containing the module's 110 decision records.

## Preserved

Single-System default behavior, all 110 protected requirements, offline operation, JSON/CRM/Word Review, images, preflight, print/PDF, Workshop handoff 1.0, SSP return 1.0, and read-only audit 0.1.

## Deliberate boundary

Declared inheritance is not synchronized. Overrides, stale-parent detection, conflict handling, consolidated CRM, governed approvals, and baselines remain later releases.
