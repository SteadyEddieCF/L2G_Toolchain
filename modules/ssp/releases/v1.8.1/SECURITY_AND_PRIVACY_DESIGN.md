# Security and Privacy Design — v1.8.1

All authoring remains local and offline with no telemetry or external calls. Imported strings are normalized to plain text and length-limited. IDs, enums, module references, requirement IDs, unique module/requirement pairs, and required rationale/source metadata are validated. Evidence is reference-only by default.

Exports carry a sensitive-system-information warning. Existing image sanitization, CSV formula protection, DOCX validation, isolated storage namespaces, recovery, backup, and reset controls remain unchanged. This release adds no cloud backend, multi-user synchronization, direct external integration, or automatic compliance decision.
